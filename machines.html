Heartbeats are written each runner loop tick when the runner is started with --remote-control. Per-machine heartbeats live at REE_assembly/evidence/experiments/runner_heartbeats/<hostname>.json; per-machine status (pre-existing) at runner_status/<hostname>.json; per-machine command queues at runner_commands/<hostname>.json. A machine is "fresh" if its last heartbeat is within 3 minutes (3x the default 60s polling interval).

Buttons enqueue commands to the target machine; the runner picks them up on its next loop tick (within ~60s). Commands work only on fresh machines because they require a running --remote-control runner to read the queue. start is not supported here -- use the explorer's local runner buttons or SSH for that.
