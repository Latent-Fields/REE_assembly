what would it actually take for “operating mode” to bind E3 across time, rather than just nudging it step-by-step?

⸻

1. The current situation (why it doesn’t bind yet)

Right now the flow is roughly:

signals → salience → operating_mode → (adapter) → E3 bias → action

That produces:

* local modulation (slight bias in scoring)
* but no persistence
* no guarantee the same mode influences the next decision

So each step is effectively:

“re-decide what kind of agent I am”

That’s the core instability.

⸻

2. What “binding across time” actually means

You need three things simultaneously:

(A) Persistence

Operating mode must exist as state, not just signal.

(B) Constraint

It must restrict the action space, not just bias scores.

(C) Hysteresis

It must be hard to switch out of, unless strong evidence accumulates.

⸻

3. Minimal viable implementation (not theoretical — actionable)

Step 1 — Create a persistent mode register

Right now operating mode is recomputed each cycle.

Instead:

mode_t = update(mode_{t-1}, salience_signals)

Not:

mode_t = f(current_signals_only)

This single change introduces temporal continuity.

⸻

Step 2 — Add hysteresis (this is critical)

Do NOT allow instant switching.

Implement something like:

* threshold to enter mode
* higher threshold to exit mode

Example conceptually:

if mode == EXPLORE:
    stay unless strong certainty signal
if mode == EXPLOIT:
    stay unless strong conflict signal

This creates:

behavioural “stickiness”

Without this, you’ll get oscillation.

⸻

Step 3 — Move from bias → constraint

Right now:

mode → small delta in E3 scoring

Instead:

mode → modifies candidate trajectory generation

Examples:

* EXPLORE mode
    * increase E2 rollout temperature
    * widen candidate set N
    * reduce residue weighting
* EXPLOIT mode
    * restrict candidate diversity
    * increase residue avoidance weight
    * shorten planning horizon

This is the key shift:

Mode changes what futures are even considered, not just how they are ranked.

⸻

Step 4 — Inject mode into hippocampal proposals

Currently hippocampus proposes trajectories relatively independently.

You want:

hippocampal_proposals = f(world_state, mode)

Not:

hippocampal_proposals = f(world_state)

This is where you get:

* mode-consistent imagination
* not just mode-consistent selection

⸻

Step 5 — Bind mode to BetaGate (policy continuity)

Right now BetaGate holds actions between E3 ticks.

Extend it:

* BetaGate also respects mode
* disallows rapid switching of policy class

This gives:

“I’m in a mode, and I act like it continuously”

⸻

Step 6 — Optional but powerful: mode-conditioned memory

Let mode affect:

* what gets written to memory
* what gets replayed

Example:

* EXPLORE → replay novelty
* EXPLOIT → replay success trajectories

This stabilises modes over longer timescales.
