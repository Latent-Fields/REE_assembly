# Processed Sources

- `legacy_tree/` is a verbatim preservation of documentation-like files discovered in the repo at rebuild time.
- Canonical docs under `docs/` are assembled primarily by extracting from `legacy_tree/` and adding minimal glue text.
- `legacy_tree_manifest.txt` captures the exact set of files mirrored from the repo; additional historical paths may remain for zero-loss preservation.
- `extracted_fragments/` contains verbatim excerpts from code or non-doc sources that reference REE concepts.
