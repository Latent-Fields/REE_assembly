Contribute Compute to REE

Donate spare GPU or CPU time to a public AI safety research project

🔍

Fully open source — read the code before running anything. Every experiment script that will run on your machine is at github.com/Latent-Fields/ree-v3/experiments. No surprises.

📄

Results only. Your machine trains small neural networks and pushes the result JSON to a public GitHub repo. No other data leaves your machine. You can inspect every output file before it's committed.

🔑

SSH key is read-only and scoped. The Docker container mounts your key read-only and uses it for one thing: git push to GitHub. It cannot access your filesystem, other keys, or any other service.

⚡

Low priority — stop any time. The runner runs at below-normal CPU priority. docker compose down stops and removes it completely. No background services, no startup entries, no persistence after removal.

👤

What this is. REE is a cognitive architecture research project exploring how AI agents can reason about ethical cost across time — not a product, not a company. Run by Daniel Golden, open Apache 2.0. Experiment results feed a public claims registry.

🔑 SSH Key Setup

The container uses your SSH key to push experiment results to GitHub. It is mounted read-only — the container cannot modify it or use it for anything else.

Confirm it works:

ssh -T git@github.com
Expected: Hi <username>! You've successfully authenticated...
No SSH key yet? Generate one here and add it under GitHub Settings → SSH and GPG keys.

⚙️ Alternative Setups

Mac — Docker CPU-only (alternative to bare-metal)

Docker on macOS runs in a Linux VM — MPS/GPU is not accessible. Bare-metal is faster. Docker is useful if you prefer not to install Python locally.

export REE_MACHINE_NAME=my-mac-name cd REE_assembly/docker docker compose -f docker-compose.cpu.yml up -d
Requires Docker Desktop for Mac.
Windows — Docker (isolated container, no Python install needed)

Requires Docker Desktop with WSL2 backend. For GPU: NVIDIA drivers ≥527.41 (Docker handles the rest automatically).

cd REE_assembly\docker docker build -t ree-runner:gpu -f Dockerfile . docker compose up -d
Or double-click REE_assembly\docker\start-ree.bat — it generates an SSH key if needed and walks you through adding it to GitHub. Explorer at localhost:8000/explorer.
Linux — bare-metal + systemd (no Docker)
pip install torch numpy scipy --index-url https://download.pytorch.org/whl/cu124 # From ree-v3/ nice -n 10 python3 experiment_runner.py --auto-sync --loop --remote-control
Persist with a systemd user unit — save as ~/.config/systemd/user/ree-runner.service:
[Unit] Description=REE Experiment Runner After=network.target [Service] WorkingDirectory=%h/ree-v3 ExecStart=/usr/bin/python3 experiment_runner.py --auto-sync --loop --remote-control Nice=10 Restart=on-failure RestartSec=30 [Install] WantedBy=default.target
Enable: systemctl --user enable --now ree-runner  |  Stop: systemctl --user stop ree-runner

🔄 How It Works

→

The runner reads from a shared experiment queue on GitHub. Before starting each experiment it writes a claim and pushes — whichever machine pushes first runs that experiment. No central server.

→

Each experiment is a short neural network training run — typically 3–30 minutes. The result is a JSON file pushed to a public GitHub repo. The project maintainer reviews results and updates the public claims registry.

→

Claims older than 6 hours are treated as stale (machine went offline) and return to the queue for other machines to pick up.

Your contribution is tracked in contributions.json — compute hours, experiment count, and estimated electricity value. Updated each governance cycle.
