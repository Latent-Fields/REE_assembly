⚙️ Start the Runner ✓ Low priority

1

From inside the ree-v3/ directory, start the runner at low CPU priority:

nice -n 10 /opt/local/bin/python3 experiment_runner.py --auto-sync --loop
nice -n 10 limits CPU scheduling priority. If your Python isn't at that path, use nice -n 10 python3 experiment_runner.py --auto-sync --loop
2

To keep it running after you close Terminal, use caffeinate + nohup:

nohup nice -n 10 /opt/local/bin/python3 experiment_runner.py --auto-sync --loop > runner.log 2>&1 &
3

To stop it:

pkill -f experiment_runner.py
1

Start at low priority from inside ree-v3/:

nice -n 10 python3 experiment_runner.py --auto-sync --loop
2

Optional: run as a systemd user unit for persistence across sessions. Save as ~/.config/systemd/user/ree-runner.service:

[Unit] Description=REE Experiment Runner After=network.target [Service] WorkingDirectory=%h/ree-v3 ExecStart=/usr/bin/python3 experiment_runner.py --auto-sync --loop Nice=10 Restart=on-failure RestartSec=30 [Install] WantedBy=default.target
Enable: systemctl --user enable --now ree-runner  |  Stop: systemctl --user stop ree-runner
1

Verify your GPU is visible to PyTorch (run in PowerShell):

python -c "import torch; print(torch.cuda.is_available()); print(torch.cuda.get_device_name(0))"
Must print True and your GPU name. If False, update NVIDIA drivers from nvidia.com/drivers.
2

Navigate to ree-v3\ and start the runner at below-normal priority. Replace my-machine-name with a short stable name for this PC (e.g. cyberpower) — used for multi-machine experiment routing:

$proc = Start-Process "py" -ArgumentList "-3.12 experiment_runner.py --auto-sync --loop --machine my-machine-name" -PassThru -NoNewWindow $proc.PriorityClass = [System.Diagnostics.ProcessPriorityClass]::BelowNormal
3

To run in the background and log output:

$proc = Start-Process "py" ` -ArgumentList "-3.12 experiment_runner.py --auto-sync --loop --machine my-machine-name" ` -RedirectStandardOutput runner.log ` -RedirectStandardError runner_err.log ` -PassThru -WindowStyle Hidden $proc.PriorityClass = [System.Diagnostics.ProcessPriorityClass]::BelowNormal Write-Host "Runner started, PID $($proc.Id)"
4

To stop it (use Ctrl+C in the PowerShell window, or):

Get-Process python | Where-Object {$_.MainWindowTitle -eq "" } | Stop-Process
GPU note: If your machine has a CUDA GPU, PyTorch will use it automatically. For a GTX 1050 Ti or similar 4GB card, experiments will run faster than CPU-only.

🔄 How Multi-Machine Coordination Works

→

The runner reads from a shared experiment queue on GitHub. Before starting each experiment, it writes a claim and pushes to GitHub — whichever machine pushes first runs the experiment. This prevents duplicate work without any central server.

→

Results are pushed back to GitHub automatically. The project maintainer reviews results and updates the claims registry.

→

Claims older than 6 hours are treated as stale (machine went offline) and become available for other machines.

Your contribution is tracked in contributions.json — compute hours, experiment count, and estimated cost (electricity + cloud-equivalent). This is updated each governance cycle.
