🚀 Quick Start

1

Request membership of the Latent-Fields GitHub organisation to access the private repos.

↗ Request Latent-Fields membership
2

Clone both repos into the same folder:

git clone https://github.com/Latent-Fields/REE_assembly.git git clone https://github.com/Latent-Fields/ree-v3.git
3

Start the runner — choose how you prefer:

Terminal (PowerShell)
cd REE_assembly\docker docker compose up -d
File Explorer
Double-click
REE_assembly\docker\start-ree.bat
Requires Docker Desktop (WSL2 backend) + NVIDIA drivers ≥527.41. No Python install needed — everything runs inside the container. Your SSH key is mounted read-only for git push.

Start the container from REE_assembly/docker/:

cd REE_assembly/docker docker compose up -d
Requires Docker Engine + NVIDIA Container Toolkit for GPU. No Python install needed inside the container. Your SSH key is mounted read-only for git push.

From inside ree-v3/, start the runner at low CPU priority:

nice -n 10 python3 experiment_runner.py --auto-sync --loop
Needs Python 3.10–3.12 + PyTorch 2.x. MacPorts path: /opt/local/bin/python3. See Setup details below if not installed.
✓

The runner starts automatically. After ~30 seconds, confirm at:

http://localhost:8000/explorer
You should see runner status Running and the current experiment name. That's it — experiments run in the background.

Runner is live. To keep it running after closing Terminal:

nohup nice -n 10 python3 experiment_runner.py --auto-sync --loop > runner.log 2>&1 &
Stop with pkill -f experiment_runner.py

⚙️ Alternative Setups

Mac — Docker CPU-only (alternative to bare-metal)

Docker on macOS runs in a Linux VM — MPS/GPU is not accessible. Bare-metal is faster. Docker is useful if you prefer not to install Python locally.

export REE_MACHINE_NAME=my-mac-name cd REE_assembly/docker docker compose -f docker-compose.cpu.yml up -d
Requires Docker Desktop for Mac.
Windows — bare-metal PowerShell (no Docker)

Install Python 3.12 from python.org (tick "Add to PATH") and Git from git-scm.com. Then:

py -3.12 -m pip install torch numpy scipy --index-url https://download.pytorch.org/whl/cu124
# From ree-v3\ $proc = Start-Process "py" -ArgumentList "-3.12 experiment_runner.py --auto-sync --loop --machine my-machine-name" -PassThru -NoNewWindow $proc.PriorityClass = [System.Diagnostics.ProcessPriorityClass]::BelowNormal
Linux — bare-metal + systemd (no Docker)
pip install torch numpy scipy --index-url https://download.pytorch.org/whl/cu124 # From ree-v3/ nice -n 10 python3 experiment_runner.py --auto-sync --loop
Persist with a systemd user unit — save as ~/.config/systemd/user/ree-runner.service:
[Unit] Description=REE Experiment Runner After=network.target [Service] WorkingDirectory=%h/ree-v3 ExecStart=/usr/bin/python3 experiment_runner.py --auto-sync --loop Nice=10 Restart=on-failure RestartSec=30 [Install] WantedBy=default.target
Enable: systemctl --user enable --now ree-runner  |  Stop: systemctl --user stop ree-runner

🔄 How Multi-Machine Coordination Works

→

The runner reads from a shared experiment queue on GitHub. Before starting each experiment it writes a claim and pushes — whichever machine pushes first runs that experiment. No central server needed.

→

Results are pushed back to GitHub automatically. The project maintainer reviews results and updates the claims registry each governance cycle.

→

Claims older than 6 hours are treated as stale (machine went offline) and become available for other machines to pick up.

Your contribution is tracked in contributions.json — compute hours, experiment count, and estimated electricity cost. Updated each governance cycle.
