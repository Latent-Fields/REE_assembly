# Summary: Lengyel & Dayan (2008) — Hippocampal Contributions to Control: The Third Way

**Entry:** 2026-03-30_arc_018_mech_033_hippocampal_dyna_lengyel2008
**Claims tested:** ARC-018, MECH-033
**Evidence direction:** supports | **Confidence:** 0.79

---

Lengyel and Dayan's 2008 NeurIPS paper is the conceptual key for understanding why REE's hippocampal architecture is computationally justified. The standard framing in computational neuroscience in 2007 was a two-system competition between model-based RL (associated with prefrontal cortex and dorsomedial striatum) and model-free RL (associated with dorsolateral striatum). Lengyel and Dayan argued for a third system: episodic control, associated with the hippocampus. Their core claim is normative — they analyse simple environments and demonstrate that episodic control should outperform both model-free and model-based RL in specific conditions: high environmental complexity, inferential noise, and especially early in learning before enough experience has accumulated for either the model or the value function to be well-calibrated.

The episodic control system they describe works as follows: stored past experiences (observation-action-outcome tuples) are retrieved by similarity to the current state, and the associated outcomes are used to directly guide action. There is no explicit model and no cached value function — just raw episode retrieval. The key insight is that this is a form of model-based planning but one that bypasses the need to learn a separable world model. The stored episodes *are* the model, implicitly. This is the Dyna architecture applied to episodic memory: Dyna (Sutton 1990) uses a learned world model to generate simulated experience for model-free updates; episodic control skips the learning step and uses actual past experience directly.

For MECH-033 and ARC-018, this is directly illuminating. MECH-033 says E2 kernels seed hippocampal rollouts. The Lengyel-Dayan framework provides the normative justification: E2's forward model extends stored episodes beyond single-step recall into multi-step rollouts. The stored hippocampal episode provides the anchor (a specific past harm trajectory); E2's forward prediction extends it forward in time (predicting what happens next given the current action-object). Together they produce ARC-018's viability map: a representation of which trajectory extensions are likely to be safe, built from a combination of episodic recall and forward simulation. This is exactly what "the third way" enables — neither pure recall nor pure model-based forward planning, but a hybrid.

The paper also analyses when each system should dominate. Episodic control is most useful early in learning (before the model or value function is reliable) and when the environment is highly variable (so past experiences are informative but a learned model would overfit). Both conditions describe REE's regime: the agent is perpetually early in learning relative to the diversity of harm configurations it might encounter, and the harm terrain is highly variable (harm events are sparse and contextual). This gives the hippocampal system a persistent rather than merely temporary advantage over pure model-based planning.

The main limitation is that Lengyel and Dayan's episodic system stores raw observations, not latent codes. REE's z_world (SD-005) is a latent representation; whether episodic control over latent spaces works comparably to episodic control over raw observations is not established in this paper. The intuition is that it should work at least as well (better compressed representations should give better generalization), but this requires the encoder (SD-005/SD-007) to produce representations that preserve the action-consequence structure that makes episodes informative for planning. If the encoder destroys this structure — as early training phases risk — episodic RL benefits would evaporate.

Confidence is 0.79: the strongest computational analogue to MECH-033/ARC-018 in the literature, with a real caveat around latent-space episodic control.
