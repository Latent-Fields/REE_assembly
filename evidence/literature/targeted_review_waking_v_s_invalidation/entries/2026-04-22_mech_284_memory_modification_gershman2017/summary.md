# Gershman, Monfils, Norman & Niv 2017 -- The Computational Nature of Memory Modification

According to PubMed ([DOI](https://doi.org/10.7554/eLife.23763)).

## What the paper does

Gershman and colleagues extend the latent-cause framework from Gershman2010 to the question of how existing memory traces get modified. The architectural move is to treat memory modification as classical associative learning operating on whichever trace is currently *eligible* for modification, where eligibility is determined by a structure-learning mechanism that segments the stream of experience into statistically distinct latent causes. New memories are formed when structure learning infers that a new latent cause underlies current observations. Old memories are modified when old and new observations are inferred to share a cause. This recasts reconsolidation -- the labile period after retrieval during which a memory can be updated -- as a Bayesian decision about whether the retrieved trace and the new observations share a generative cause.

## Why it matters for V_s invalidation

For us this paper does the architectural job of bridging waking V_s invalidation (MECH-284) and sleep-priority replay (MECH-285). The structure-learning operation -- "is this new observation generated by the same latent cause as the previously anchored one?" -- is exactly the operation MECH-269 needs to decide whether to fire the anchor reset. And the same operation, applied during sleep replay, is what determines which schema gets consolidated and which gets superseded by a new one. The coupling between waking invalidation and sleep priority falls out naturally from this framework rather than being an additional posit.

The architectural reading is: during waking, the structure-learning mechanism receives evidence (mediated by DA generalized PE, OFC state labelling, alEC/DG interference detection) that the current latent cause is becoming less probable. The accumulator is the Bayesian update over time. When the posterior over a competing cause crosses threshold, MECH-269 fires anchor reset and the new latent cause becomes operative. The previous cause's trace is *not* erased -- it is marked for sleep replay, which is the operation by which the regional schema database gets updated to reflect the new structure. This matches MECH-285's framing of sleep replay as the consolidation that updates V_s for stored schemas.

## Architectural implications

This paper also gives us a principled way to think about the coupling question (architectural question 4 in the spec). The coupling between waking invalidation load and sleep replay is not an arbitrary posit -- it follows from the structure of the Bayesian update. Waking invalidation generates a posterior that some latent causes have become less probable; sleep replay is the operation that propagates that posterior to the long-term schema database. High-mismatch days should produce both more waking invalidation events and more replay-priority for the affected schemas, consistent with the clinical observation that high-distress days produce both intrusive next-day cognition and elevated REM%.

## Clinical resonance

The framework's failure modes map cleanly to clinical phenomena. Over-segmentation (inferring new latent causes too readily) produces parallel uncombined memory traces -- a candidate substrate for dissociative phenomena, where the same situation gets stored as multiple unconnected episodes. Under-segmentation (failing to infer new causes when warranted) produces traumatic-memory modification that incorporates new triggers -- a candidate for PTSD generalisation, where the original threat schema gets extended to encompass new and inappropriate stimuli. Both failure modes are V_s-architecture-relevant: the first is over-active accumulator firing, the second is under-active accumulator failing to cross threshold.
