# Shenhav, Botvinick, Cohen 2013 — Expected Value of Control theory of dACC (MECH-268)

According to PubMed ([DOI](https://doi.org/10.1016/j.neuron.2013.07.007)), Shenhav, Botvinick, and Cohen's Neuron 2013 paper is the foundational statement of the Expected Value of Control (EVC) framework for dorsal anterior cingulate cortex function. The proposal is that dACC integrates three factors — the expected payoff from a controlled process, the amount of control required to achieve that payoff, and the cost of cognitive effort — into a single value signal that specifies the currently optimal allocation of control. This framework unifies a diverse range of dACC findings (reward processing, monitoring, cognitive control, action selection) under one normative architecture.

For MECH-268 the EVC framework provides the normative motivation for the saturation claim. MECH-268 asserts that the dACC-analog pe signal in SD-032b should cap and habituate under repeated identical outcomes — that pe_saturated = f_sat(pe_raw, outcome_history) where f_sat bounds growth and introduces habituation. The EVC framework explains why this should be so: if a controlled process is producing repeated identical outcomes, the expected payoff of continued control drops (nothing new is learned, no new value is gained), and therefore the EVC signal — and the control allocation it drives — should drop. Saturation is what a value-optimising dACC should do when faced with repetitive, non-informative outcomes. MECH-268's saturation is therefore normatively licensed by EVC, not an arbitrary engineering patch.

The honest caveat I record is that EVC is a framework, not a specific mechanism. Multiple implementations can track EVC: a bounded pe with outcome-identity habituation (MECH-268's choice); an unbounded but well-calibrated pe signal modulated by an external expected-payoff channel; a pe signal that never saturates but whose downstream control allocation does. EVC does not discriminate among these. MECH-268's commitment to bounded + outcome-identity-buffer is one parameterisation; the V3-EXQ-463 experiment will probe whether the bounded-pe implementation is the right one or whether a more EVC-faithful implementation (external modulation rather than internal saturation) is preferable.

For governance purposes the interesting observation is that MECH-268 pairs with SD-034 (closure operator). SD-034 is explicit closure — dACC pe is reset per rule-episode when a closure token fires. MECH-268 is implicit closure — dACC pe caps under repetitive outcomes even without explicit closure. Both are EVC-consistent in the sense that both drop the control-allocation signal when further control offers no value. The combination — explicit closure on successful completion, implicit saturation under stuck repetition — is a sensible governance architecture for a loop that must both respond to completion signals and degrade gracefully when those signals are missing. The EVC framework gives a single normative account of both.

Confidence is set at 0.68. Source quality is very high (Neuron, canonical EVC paper). Mapping fidelity is moderate because EVC is a normative framework and MECH-268's specific saturation commitment is an implementation choice that is consistent with but not derived from EVC. Transfer risk is moderate: EVC is a human/primate dACC theory, while SD-032b is REE's smaller dACC-analog without full EVC machinery. I record `evidence_direction: mixed` because EVC supports the direction of MECH-268 (control should drop under repetitive outcomes) while being agnostic about the specific implementation path (bounded pe vs external modulation).
