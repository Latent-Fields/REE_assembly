# Summary: Mattar & Daw (2018) — Prioritized memory access explains planning and hippocampal replay

## What the paper did

Mattar and Daw address a question that is both empirically tractable and theoretically profound: of all the states an animal has visited, which ones actually get replayed during rest? Their answer is normative — they derive which states should be replayed on the basis of expected utility, and then show that this prescription matches empirical observations about the content of hippocampal replay that had previously seemed unrelated. The core metric is a utility function with two factors: 'need' (how likely is the agent to visit this state in the near future, given current policy?) and 'gain' (how much would future decision quality improve if the value estimate of this state were updated?). States scoring high on both get replayed first. The model is implemented as a sequential, nonlocal reactivation algorithm and validated against rodent replay datasets, including reverse replay after reward, prospective replay before departure, and reward-magnitude-sensitive replay content.

## Key findings relevant to ARC-036

The paper's central result — that replay content reflects decision-relevant value structure rather than recency or proximity — is directly relevant to ARC-036 in a specific way that differs from its relevance to ARC-007. For ARC-007, the relevance is about path-level prioritization: paths through high-residue terrain get preferentially replayed because harm salience creates high gain. For ARC-036, the relevance is about dimensional selectivity: if hippocampus encodes a multi-dimensional valence terrain (wanting, liking, harm, surprise), then the prioritized replay process would selectively consolidate different dimensions depending on which dimension currently has the highest utility.

Consider the behavioral context dependency. An animal in a food-deprived state, where wanting-salience dominates, would have high utility for replaying food-associated states — not because those states are spatially central but because wanting-dimension gain is currently high. The same animal in a dangerous environment has high utility for replaying harm-salient states. This context-sensitivity of replay — which dimensions get consolidated when — is what SD-012 (homeostatic drive scaling) is attempting to implement biologically. Mattar and Daw's framework provides the normative justification for why drive-state-gated replay is computationally appropriate: different drive states should gate different dimensions of the valence terrain into active consolidation.

A second relevant finding is the treatment of the successor representation (SR) variant. When replay is implemented over the SR rather than over direct state values, replay naturally organizes around state transitions that matter for future navigation — which is closer to the predictive-map framework of Stachenfeld et al. (2017). For ARC-036, the SR variant is relevant because the terrain map is not a static value map but a dynamic representation of future state occupancy — a representation that must track the multi-dimensional valence of where the agent is likely to go, not just where it has been.

## REE translation

For ARC-036, the Mattar-Daw framework provides the consolidation mechanism that a multi-dimensional hippocampal valence terrain would require. The claim is that hippocampus does not merely encode instantaneous valence but maintains a coherent, plannable representation of the valence landscape — the terrain map. That map must be actively maintained through replay because state space is large, temporal discounting erodes old associations, and the relevant valence dimensions shift with drive state. Mattar and Daw explain how this maintenance would work: the replay system continuously identifies which states, along which dimensions, are most decision-relevant given current policy and expected future trajectory, and prioritizes those for consolidation.

This mapping has a specific implication for SD-012 (homeostatic drive) and its relationship to ARC-036: if drive_level modulates the utility calculation — making hunger increase the need-gain product for food-associated states, threat increase it for harm-salient states — then drive state is not merely a scaling factor on the terrain but a selector of which terrain dimension gets active replay consolidation. This is a richer account of drive-state-terrain interaction than SD-012 as currently specified, and it is one that Mattar and Daw's framework naturally supports.

Note that this entry and the ARC-007 Mattar-Daw entry have different mapping foci. The ARC-007 entry maps onto path-level prioritization: high-residue paths get replayed preferentially. This entry maps onto dimension-level prioritization: whichever valence dimension is decision-relevant in the current drive state gets consolidated preferentially. The two mappings are complementary — both operate through the same need-gain mechanism but at different levels of the terrain representation.

## Limitations and caveats

The central limitation is the scalar value assumption. The model has a single utility function over a single value dimension (expected reward). Extending it to a multi-dimensional valence terrain — where wanting, liking, harm, and surprise each have separate utility calculations — would require either: (a) defining a multi-dimensional utility function that integrates across dimensions with drive-state-dependent weights, or (b) running separate replay queues for each valence dimension with inter-queue competition for replay bandwidth. Neither extension is developed in the paper, and both are non-trivial. The paper provides the normative skeleton; the multi-dimensional flesh must be added.

A second limitation is the absence of any empirical paradigm where distinct valence dimensions are in conflict. All of the paper's validation scenarios involve a single reward signal. Whether hippocampal replay actually tracks multiple separable valence dimensions under naturalistic conditions — let alone whether it prioritizes them independently — is not addressed. Third, the temporal dynamics of replay in the model are stylized: the 'rest period' is a single planning episode. In real animals, replay occurs across multiple timescales (SWR-associated replay, REM replay, quiet wakefulness replay), and it is not clear that the same utility function governs all of them.

## Confidence reasoning

Confidence is set at 0.78 — somewhat lower than the ARC-007 entry (0.74) for two reasons of different valence. First, the mapping is more indirect here: ARC-007's terrain residue maps cleanly onto 'value' in the Mattar-Daw framework because residue is an aversive valence. For ARC-036's multi-dimensional terrain, the extension requires assuming that each dimension has its own utility calculation, which is a larger theoretical step. Second, the multi-dimensional terrain is exactly what ARC-036 is claiming — it is not yet established — so the paper supports the claim by providing a mechanism that would work if the terrain is multi-dimensional, but it does not demonstrate that the terrain is multi-dimensional. The confidence reduction reflects this circularity: the paper supports the mechanism but not the premise. Despite this, the paper is essential to the evidence base because no other framework provides as explicit a mechanism for how a valence terrain would be actively maintained by replay.
