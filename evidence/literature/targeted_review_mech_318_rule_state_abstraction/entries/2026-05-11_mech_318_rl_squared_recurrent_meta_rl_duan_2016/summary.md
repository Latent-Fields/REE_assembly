# Duan et al. 2016 -- RL² : Fast RL via Slow RL

The RL² paper ([arXiv:1611.02779](https://arxiv.org/abs/1611.02779)) makes a clean move: rather than designing a fast reinforcement-learning algorithm by hand, encode the algorithm in the weights of a recurrent neural network and learn those weights with a slow general-purpose RL algorithm. The RNN receives the standard inputs (observations, actions, rewards, termination flags); its hidden-state activations at any moment represent the state of the fast algorithm on the current MDP. Crucially, the hidden state is preserved across episodes within a task instance, so the agent can accumulate evidence about the MDP over multiple episodes. The result is that a learnt RL algorithm runs inside a learnt RNN -- hence the squared.

For MECH-318 this is the foundational AI-tradition anchor for cross-episode hidden-state continuity -- the W5 property of the absorption check (2026-05-10). The absorption check identified W5 as NOT ABSORBED by any current REE substrate: E1 LSTM resets per episode (deliberately, to anchor the per-episode world-prediction objective); SD-033a rule_state buffer resets per episode (Choice A2 in the design docstring; cross-episode carry-over explicitly deferred to V4); MECH-269 anchor sets reset per episode (active and inactive anchors both cleared). Cross-episode continuity is not present in V3. Adding it to any of these substrates is architecturally significant and conflicts with the existing per-episode state-machine commitments that anchor the rest of the agent's per-episode reset semantics.

The substrate-design implication is that the RL² flavour of MECH-318 is straightforwardly V4-scope. The within-V3 portion (W1+W3+W4 covered by SD-033a + ARC-062 Phase 1 + planned Phase 3 wiring) does not include W5. The behavioural-signature implication is that a V3 instantiation of MECH-318 will be able to display within-episode rule-state adaptation but not cross-episode evidence accumulation. If empirical validation on the multi-rule-context substrate that V3-EXQ-543c-successor needs shows that within-episode adaptation alone is sufficient, MECH-318 retires as superseded by the SD-033a + ARC-062 cluster and the W5 / RL² question moves to V4. If the validation shows that cross-episode evidence accumulation is empirically required, MECH-318 promotes to active and motivates either a dedicated substrate landing or a substrate-architecture extension that breaks the per-episode reset.

The case for treating W5 as V4-scope is that the existing per-episode reset is architecturally load-bearing for REE's state-machine. Anchor-set partition timing, sleep / wake transitions, harm-record accumulation, ghost-goal bank cleanup -- all of these are gated on the per-episode boundary. Decoupling MECH-318's hidden state from this boundary while preserving the others would require a substrate-architecture pass that V3 is not designed for. The cleaner architectural commitment is to accept that the cross-episode meta-RL portion of the rule-state abstraction is a V4 / strong-reading-of-ARC-063 question.

The case for treating W5 as V3-scope is weaker. The only way W5 could be V3-tractable is if REE were redefined to have a meaningful 'task instance' that spans multiple episodes -- e.g. a sequence of episodes under the same rule-regime that the agent should treat as a single learning context. This would require substrate-design commitments beyond what V3 currently has and is probably the wrong place to make them. V3's task structure is single-episode-per-instance by design.

The mapping caveat I would not want to bury: RL² was designed for AI tasks where 'task instance' is a well-defined construct (a specific MDP within a family) and 'cross-episode within a task instance' is meaningful (multiple episodes within the same MDP instance, separated by environment resets but unified by shared dynamics and reward structure). REE's behavioural substrate has different semantics: each episode is a fresh start, and the analog of 'task instance' would require a substrate-level commitment to multi-episode coherent task structure that V3 does not currently have. The transfer from RL²'s MDP-family paradigm to REE's behavioural-substrate paradigm requires substrate-design work that this lit-pull does not by itself resolve.

I rate this confidence 0.78. Source quality is moderate (arXiv preprint, but highly-cited and foundational). Mapping fidelity is good for the W5 property specifically. Transfer risk is elevated because the cross-episode-continuity property may simply be V4-scope for REE given the existing per-episode reset architecture. The RL² flavour is the right framing for what W5 names; the substrate-design implication is that W5 is V4-deferred and the within-V3 portion of MECH-318 absorbs into existing substrates as the absorption check already settled.
