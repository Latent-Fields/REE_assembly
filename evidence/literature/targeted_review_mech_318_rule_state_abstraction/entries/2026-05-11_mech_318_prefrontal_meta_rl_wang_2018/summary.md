# Wang et al. 2018 -- Prefrontal cortex as a meta-reinforcement learning system

According to PubMed ([DOI](https://doi.org/10.1038/s41593-018-0147-8), PMID 29760527), Wang and colleagues propose a reframing of what prefrontal cortex is doing that has substantial consequences for how to think about rule-state abstraction. The framework is meta-reinforcement learning, drawn from advances in deep RL: a recurrent network's weights are slowly trained by an outer-loop algorithm (in the biology, dopaminergic prediction-error TD learning) across many task instances, and the slow weight learning installs an inner-loop learning algorithm in the network's hidden-state dynamics. Within any single task instance the agent learns through hidden-state dynamics -- without weight updates -- because the meta-learning algorithm is already running. The proposal is that PFC IS the substrate where this happens. The hidden state at any moment encodes which task the agent is currently in; the recurrent dynamics implement the appropriate within-episode adaptation; the dopamine system trains the weights across episodes to install the algorithm.

For MECH-318 this is the load-bearing primary anchor. The 2026-05-10 absorption check (`docs/architecture/mech_318_absorption_check.md`) audited the existing ree-v3 candidate substrates -- E1 LSTM, SD-033a LateralPFCAnalog rule_state buffer, MECH-269 anchor sets -- against the five Wang 2018 properties (W1 recurrent topology, W2 trained across tasks, W3 hidden state encodes task identity, W4 biases action selection, W5 cross-episode hidden-state continuity) and reached verdict B: partially absorbed. The within-V3 portion (W1+W3+W4) is covered by SD-033a + ARC-062 Phase 1 + planned Phase 3 wiring; the W2 multi-task-training and W5 cross-episode continuity properties are NOT absorbed. This lit-pull confirms via the literature what the absorption check identified architecturally: the gaps are real and they correspond to load-bearing functional properties that Wang et al. anchor.

The substrate-design implication is sharp. If MECH-318 is the right framing for what REE needs at the rule-state-abstraction layer, the substrate-enrichment work to close the W2 + W5 gaps becomes essential. W2 requires a multi-task training distribution: SD-054 single-context is insufficient; the substrate has to be enriched to provide multiple rule-contexts that the agent encounters across episodes, with weight learning shaped by the family. W5 requires cross-episode hidden-state continuity: the current SD-033a per-episode reset architecture has to be modified (which would conflict with the existing per-episode state-machine commitments) or accepted as a V4 deferral.

The architectural sequencing reading: V3 may be able to instantiate the within-episode portion of MECH-318 (W1+W3+W4) without W5, on a multi-rule-context substrate that provides W2. This is what V3-EXQ-543c-successor would test. If the within-episode portion produces the rule-state-adaptation signature on the multi-rule-context substrate, MECH-318 retires as superseded by the SD-033a + ARC-062 cluster. If it does not, MECH-318 promotes to active and motivates a dedicated substrate landing in V4.

The case for cleanly framing the gaps as W2 (multi-task training distribution) and W5 (cross-episode continuity) rather than as 'MECH-318 needs a new substrate' is that the within-V3 substrates have already done significant work toward the rule-state-abstraction role. The Pull 1 substrate audit lists SD-033a as recurrence-as-EMA (W1 partial), ARC-062 GatedPolicy as per-tick rule-context discriminator (W3 per-tick), and SD-033a compute_bias + ARC-062 heads as the downstream action-selection bias (W4). What is missing is the training signal that would shape the substrate into a content-is-rule-state encoder -- and that requires the multi-rule-context training distribution that W2 names.

The mapping caveat I would not want to bury: Wang et al. operationalise their account on deep-RL benchmark tasks (Harlow object-discrimination task, two-step task, bandits with stationary/non-stationary structures). The recurrent component is an LSTM trained end-to-end by REINFORCE or A2C. REE's MECH-318 has to operate within a substrate where the recurrent component (SD-033a LateralPFCAnalog) is currently a gate-modulated EMA rather than an LSTM, where the training signal for the bias head is currently frozen (Choice A2 in the SD-033a docstring), and where the multi-task training distribution does not yet exist. The transfer from Wang 2018 architecture to MECH-318 implementation is plausible but requires committing to substrate enrichments that V3 does not currently have. The absorption check already names these gaps; this lit-pull confirms them.

What does failure look like? If MECH-318 is implemented as a recurrent network without rule-context-shaping training signal, the hidden state will encode some compressed representation of the recent observation history but not specifically the rule-context (which is the W2 + W3 content failure). If the multi-rule-context training distribution is in place but the substrate retains the per-episode reset, the agent will adapt within an episode but not accumulate evidence across episodes (the W5 failure). The two failure modes are decomposable, and the V3 substrate-design pass can sequence them: build W2 first (multi-rule-context substrate; tractable for V3) and treat W5 as V4-deferred.

I rate this confidence 0.84. Source quality is high (Nat Neurosci, top-tier venue, DeepMind + Botvinick lab). Mapping fidelity is strong for the rule-state-encoding role -- the five Wang properties are the right architectural decomposition for what MECH-318 needs. Transfer risk is moderate because the deep-RL benchmark paradigm is one step removed from REE's PFC-analog architecture and the substrate-enrichments required (W2 + W5) are not yet in V3. This is the load-bearing primary anchor for the cluster.
