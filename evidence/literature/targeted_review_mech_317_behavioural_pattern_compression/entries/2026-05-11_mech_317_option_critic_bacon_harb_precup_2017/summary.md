# Bacon, Harb & Precup 2017 -- The option-critic architecture

The 2017 AAAI option-critic paper ([DOI](https://doi.org/10.1609/aaai.v31i1.10916), [arXiv 1609.05140](https://arxiv.org/abs/1609.05140)) takes on a problem the Sutton-Precup-Singh 1999 options framework left open: how does the agent actually discover options autonomously? The 1999 paper provided the computational definition (the option triple) and the convergence guarantees, but the options themselves were either hand-engineered or discovered via methods that required sub-goals, auxiliary rewards, or other specifications outside the scalar reward signal. The option-critic contribution is to derive policy-gradient theorems for the options framework itself, allowing all three components of an option (intra-option policy, termination function, and the policy-over-options) to be learned end-to-end from the same scalar reward signal that drives standard policy learning.

This is the canonical algorithmic anchor for the 'gradient option discovery' implementation flavour of MECH-317. The architectural commitment under this flavour is to a fully-differentiable substrate where each chunk's internal structure is a learned function of network parameters. The training signal is the same reward signal that drives standard policy learning -- no separate option-discovery objective is required. The implementation is parsimonious and elegant.

The practical record of option-critic is more mixed than the theoretical elegance suggests. Two failure modes are documented in the post-2017 literature and partially anticipated in the original paper:

- **Option collapse**: all options converge to identical behaviour, so the option-level abstraction adds nothing. This happens when the termination-function gradient is strong enough to collapse meaningful inter-option differences.

- **Option degeneracy**: termination probabilities converge to 'terminate-everywhere' (zero-length options, equivalent to flat policy) or 'terminate-nowhere' (one long option, also equivalent to flat policy). The Harb et al. 2017 follow-on paper (\"When Waiting Is Not An Option: Learning Options with a Deliberation Cost\") explicitly addresses this by adding a deliberation cost that penalises premature termination; the fix is necessary but not sufficient.

Both failure modes are consequences of the gradient-discovery substrate not having external constraints on what options should look like. For REE's MECH-317 this is the key architectural caution. If MECH-317 is implemented as pure option-critic on top of the existing E3 policy network, the same failure modes will appear -- chunks will collapse to identical behaviour or to degenerate length distributions. The substrate-design pass for MECH-317 should not be pure gradient option discovery. The recommended architecture is closer to a hybrid: gradient option-critic as the inner-loop learning rule for option-component parameters, wrapped in substrate-driven trigger machinery (V_s drops, MECH-269 anchor boundaries, repetition+outcome-consistency triggers per the ARC-071 lit-pull R1 verdict) that constrains when and where options form.

This hybrid is consistent with what the chunk-formation operator MECH-323 (registered today in a parallel session) actually specifies. MECH-323's trigger is repetition+outcome-consistency, NOT pure reward-gradient; the option-component parameters can be learned by option-critic-style gradients but the option-EXISTENCE decision is driven by the substrate-level repetition signal. This avoids the option-collapse and option-degeneracy modes that pure option-critic exhibits.

The mapping caveat I would not want to bury: option-critic was designed for deep-RL benchmark tasks (Atari, four-rooms, simulated control) where the reward signal is dense and directly informative. REE's MECH-317 has to operate on a substrate where chunk-formation is partly substrate-driven rather than only reward-gradient driven. The gradient-discovery flavour may be appropriate as the inner loop but probably needs to be wrapped in the substrate-driven trigger machinery rather than replacing it. Choosing option-critic-style gradient discovery as the SOLE implementation flavour for MECH-317 risks reproducing the documented failure modes; combining it with substrate-driven triggers avoids them.

I rate this confidence 0.78. Source quality is reasonable (AAAI, foundational team continuing the 1999 Sutton-Precup-Singh lineage). Mapping fidelity is good for the implementation-flavour question. Transfer risk is moderate because option-collapse failure modes are documented and the deep-RL benchmark paradigm is one step removed from REE's behavioural-trajectory substrate. The framework deserves serious consideration in the substrate-design pass as the inner-loop learning rule -- as long as the substrate-design also commits to external trigger machinery rather than relying on the gradient signal alone.
