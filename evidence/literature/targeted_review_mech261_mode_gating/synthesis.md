# MECH-261 mode-conditioned offline-write gating -- lit-pull synthesis

**Session:** litpull-mech261-mode-gating-2026-04-20T06
**Complements:** targeted_review_systems_consolidation_waking_propagation (Frankland/Bontempi;
Tambini & Davachi; Peyrache; Carr; Rothschild) which covered the *waking-consolidation
propagation* leg. This pull extends to the *mode-gating rule* leg -- what determines
whether, when, and where an offline write actually lands.

## The argument in one paragraph

MECH-261 abstracts the offline write-target decision as a dict-keyed gate multiplied by
a per-mode probability vector. The biology supporting that abstraction is tighter and
more specific than the abstraction itself. In SWS, the write gate is not a scalar per
target -- it is realised by a phase-coupled carrier-rhythm product (slow oscillation +
thalamic spindle + hippocampal ripple) in which each component licenses the others.
Driving spindles *in-phase* with the cortical slow-oscillation up-state boosts
consolidation; driving them *out-of-phase* does not, despite matched spindle counts
(Latchoumane 2017). Reinforcing the endogenous SWR-delta-spindle coupling with
closed-loop stimulation in rats selectively reorganises mPFC ensembles and rescues
otherwise-failed spatial memory (Maingret 2016). The locus of the coupling has an
anatomical correlate: medial frontal cortical atrophy in human older adults disperses
SO-spindle coupling and predicts overnight forgetting (Helfrich 2018). In REM, the
gate is entirely different -- theta generated by medial-septal GABAergic neurons is
the carrier, and optogenetically silencing it *only during REM* abolishes
consolidation while identical silencing outside REM is inert (Boyce 2016). Klinzing,
Niethard and Born (2019) provide the synthesising framework: modes are functionally
dissociable channels with dissociable carrier rhythms and dissociable plasticity
outcomes, not variants of a single gate scaled differently.

## Implications for the REE implementation of MECH-261

1. **Per-mode gate weights in the write-gate registry are load-bearing, not cosmetic.**
   The default SWS weights and default REM weights should not collapse into each
   other. SD-032a's `DEFAULT_GATE_WEIGHTS` captures this already; the lit supports
   keeping them separate rather than homogenising in a future refactor.

2. **Carrier rhythm is the biological realisation of the gate in SWS and REM.**
   The current V3 implementation models the mode as a soft probability scalar per
   tick. This is the minimal viable abstraction. A V4 implementation that wants to
   capture the Latchoumane / Boyce causal structure will need per-mode internal
   clocking (SO-spindle-ripple timing for SWS; theta timing for REM), not just a
   scalar. Flag for V4 SD-033 detailing.

3. **Gate locus and write target may overlap anatomically.** Helfrich's MFC atrophy
   result hints that the substrate that *computes* the gate and the substrate that
   *receives* the gated write overlap in medial frontal territory. A V4
   implementation that cleanly separates "gate computer" from "gate consumer" into
   physically distant nodes would diverge from this; single-substrate implementations
   in which SD-033a both produces and consumes the gate are closer to the biology.
   Record in sd_033_pfc_subdivision_architecture.md when MECH-261's staged
   implementation is next revisited.

4. **The REM within-mode target selection is an open biological question.** Boyce
   establish mode-separability for REM but not target-selectivity within REM. The
   current REE default (REM mode routes writes to residue replay + HC viability
   rather than a specific SD-033 subdivision) reflects that gap honestly. Do not
   over-commit the REM weights until target-selective REM evidence exists. A future
   Prong D lit-pull on REM theta-phase reverse-replay subpopulations (Poe 2017 and
   successors) would be the natural next step.

5. **Out-of-phase stimulation is an existence-proof falsification target.** An
   experimental arm that induces SWS-mode writes outside the phase window (via
   coordinator tick that ignores the carrier rhythm) should fail to consolidate even
   with the mode signal active. MECH-261's current V3 abstraction cannot detect this
   because it has no carrier rhythm, but the failure mode is the signature that the
   V3 abstraction is a reduced model rather than the final one.
