# O'Reilly and Frank 2006 — PBWM frontostriatal gating (MECH-266)

According to PubMed ([DOI](https://doi.org/10.1162/089976606775093909)), O'Reilly and Frank's PBWM paper is the foundational computational model of basal-ganglia-based dynamic gating of prefrontal working memory. The model implements opposing D1 (Go) and D2 (NoGo) pathways that jointly control when PFC content is updated vs maintained. The actor-critic architecture combines an external reinforcement signal (critic) with a gating actor (BG), which learns to open or close the update gate on a trial-by-trial basis. PBWM is notable for solving the temporal credit-assignment problem without explicit backpropagation-through-time by using learned gating signals to select which PFC representations to update.

For MECH-266 the paper is an architectural reference point rather than direct support. MECH-266 asserts that SD-032a's mode register should be extended with explicit (enter_threshold, exit_threshold) pairs per operating mode, making the mode gate a classic Schmitt trigger. PBWM establishes a related but different architectural principle: the WM gate has opposing Go and NoGo pathways that produce asymmetric dynamics. When D1-tone is high, updates are favoured; when D2-tone is high, maintenance is favoured. This is asymmetric hysteresis at the item-level WM gate.

The mapping is at the architectural-principle level. Two things carry over from PBWM to MECH-266: first, biological gating between stable-hold and flexible-update is implemented via opposing controllers, not a single symmetric threshold; second, the balance point between those controllers is modifiable by tonic inputs (dopamine in PBWM) that reflect context. What does not carry over cleanly is the per-mode parameterization. PBWM has a single gate (with D1/D2 opposition); MECH-266 has four operating modes, each with its own (enter, exit) pair. PBWM does not naturally support per-mode parameters — the model would implement mode-specific asymmetry through a context-dependent tonic DA setting, not through per-mode gate parameters.

The caveat here is significant and worth flagging in governance. MECH-266's implementation commitment — explicit per-mode (enter, exit) pairs — is an engineering choice that departs from the PBWM architectural pattern. A more PBWM-faithful REE implementation would be a single mode-gate with a tonic-gain parameter that varies with context. Whether REE's per-mode parameterization is equivalent, better, or worse than the PBWM-style gain-modulated single gate is an open question. The V3-EXQ-464 (competing goals) experiment will probe this directly: if a single gate with context-gain reproduces the behavioural signatures MECH-266 predicts, the per-mode parameterization is unnecessary.

Confidence is set at 0.60. Source quality is high (foundational PBWM paper). Mapping fidelity is moderate because of the single-gate vs per-mode architectural mismatch. Transfer risk is substantial because PBWM is about item-level WM updating, while MECH-266 is about operating-mode transitions. I record `evidence_direction: mixed` rather than `supports` because the paper endorses the general principle of asymmetric hysteresis at the gate level while implementing it with a different parameterization than the one MECH-266 proposes.
