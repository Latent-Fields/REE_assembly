## Binary resolution (Part A, 2026-05-17, read-only code trace)

**Question (from the deferred full audit):** Are MECH-314b's / MECH-314c's broadcast phases wired into the *pre-Phase-3 establishing gradient* — i.e. do they actually shape persistent structure (gated-policy head / residue centers / z_goal) during the infant curriculum's pre-Phase-3 window — or do they act only after the curriculum's plasticity-open phase / not gradient-couple at all?

**Verdict: NO.** Decidable from code; no experiment needed.

Two independent code facts, the second decisive:

1. **No autograd path (applies to the whole curiosity family, incl. MECH-111 — so not the discriminator on its own).** `StructuredCuriosity` is "pure-arithmetic, no learned parameters, no nn.Module" (structured_curiosity.py:192). `_compute_uncertainty` returns `float(e3._running_variance)` (structured_curiosity.py:444 — a detached Python float); `_lp_ema` is a Python float (line 363). The 314b/c contributions are `float * torch.ones(K)` (lines 347-349, 365-367). The call site is wrapped `with torch.no_grad():` (agent.py:3190), inputs `.detach()`-ed (agent.py:3192), output detached (agent.py:3198, 3312), and fed only as `score_bias=dacc_score_bias` into `self.e3.select(...)` (agent.py:3337) — candidate *selection*, not a weight update. There is **zero** autograd edge from 314b/c to the gated-policy head, residue centers, or z_goal. (Note: MECH-111 — e3_selector.py:610-611 `score -= novelty_bonus_weight * _novelty_ema` — is *also* a no-grad selection bias, not a literal gradient; the register's HIGH classification of MECH-111 rests on the **indirect experience-distribution / coverage-seeding** pathway, so "it's no_grad" alone does not discriminate 314b/c from MECH-111.)

2. **Decisive: 314b/c are never active in any infant-curriculum pre-Phase-3 window (so the indirect coverage-seeding pathway that makes MECH-111 HIGH cannot operate for them).**
   - `InfantCurriculumScheduler.config_overrides()` (infant_curriculum.py:167-189) returns, for every phase 0/1/2/3, **only** `novelty_bonus_weight` (MECH-111), `residue_scale_factor`, `offline_integration_frequency`. It **never** sets `use_structured_curiosity`, `use_curiosity_uncertainty`, or `use_curiosity_learning_progress` in any phase. The "Phase 2: structured curiosity dominant" docstring (infant_curriculum.py:163) is aspirational and unbacked by `_table`; it refers to the MECH-111 `novelty_bonus_weight` regime, not the MECH-314 module. So the curriculum itself never phases 314b/c at all — there is no "pre- vs post-Phase-3" distinction for them in the curriculum.
   - Every experiment script that drives `InfantCurriculumScheduler` was checked: `v3_exq_586_gap9_curriculum_validation_v3.py`, `v3_exq_591_isef005_curriculum_vs_flat_v3.py`, `committed_mode_curriculum.py`, `v3_exq_543g_arc062_outcome_coupled_falsifier.py` (the failed-handoff run), `v3_exq_543b_arc062_phase3_optimized_falsifier.py` — **none set any curiosity flag** → `use_structured_curiosity` defaults False (config.py:1588) → `StructuredCuriosity` is never instantiated (`self.curiosity is None`, agent.py:592). 314b/c contribute literally nothing in those runs.

## Shape assessment (one paragraph)

INV-075 remediation is a **small-N program, not a 1-fix problem, but it is far closer to N=2-4 than to N=10.** The substrate's diversity stack is overwhelmingly F-*independent* by construction: MECH-313 (constant floor), MECH-314a (residue-coverage), MECH-320 (reward-rate EWMA), MECH-260 (action FIFO), MECH-295 (drive·proximity), the z_goal pathway, serotonin tone, and the conditioned-safety store all either escape the antecedent (option (i), success-decoupled) or are already in a durable-maintenance handoff (option (iii)) — they are LOW. The genuinely exposed signals are the **F-error-coupled novelty/curiosity family**: MECH-111 (firmly HIGH, and notably *outside* the Q-052 audit's scope — a finding this pass adds), and MECH-314b/c (HIGH-pending on one shared load-bearing question). The **543 gated-policy case is representative, not idiosyncratic**: it is the INV-074 *competitive* specialization of the same parent failure (a self-extinguishing establishing signal — there the diversity gradient out-competed by F-dominance — building a load-bearing structure with no lock), and the lock it is receiving (MECH-334 crystallization) is exactly the option-(ii) arm the MECH-111/314b/c signals also lack. The structural similarity means a single crystallization/handoff design pattern, once validated on the 543 lineage, is the likely template for the F-error-coupled novelty signals too — i.e. the program is N-fix in *count of signals* but plausibly **1-pattern in remediation design**, contingent on whether MECH-111 wants a LOCK (share the 543 crystallization pattern) or an option-(i) re-grounding onto an F-independent novelty source (the Q-052 audit's alternative recommendation). **UPDATE 2026-05-17:** the MECH-314b/c load-bearing binary is now resolved **NO** (Binary resolution section) — they are dormant/default-off and never active in any infant-curriculum pre-Phase-3 window, so the headline is fixed at **1 (MECH-111 only)**, not 1-to-3. The program is therefore N=1-firm in the present substrate (plus the in-flight 543 lock), with 314b/c demoted to a contingent-future guardrail rather than an open gap.
